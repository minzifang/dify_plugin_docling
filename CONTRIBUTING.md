# Contributing

Thanks for considering a contribution.

## Development

1. Create a branch from the latest main branch.
2. Keep changes focused on one issue or feature.
3. Run the local checks from `README.md`.
4. Update `README.md`, `README.zh-CN.md`, and `CHANGELOG.md` when behavior changes.

## Pull Requests

Please include:

- What changed.
- Why it changed.
- How it was tested.
- Any Dify or Docling Serve version assumptions.

## Documentation

The English README is the primary document. Keep `README.zh-CN.md` in sync for user-facing behavior, configuration, and troubleshooting changes.

## Generated Files

Do not commit generated `.difypkg` files, signed packages, local caches, or private signing keys. Release packages should be uploaded as GitHub Release assets.
